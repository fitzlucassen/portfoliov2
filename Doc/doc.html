Configuration

Tout d'abord, il faut configurer votre application. Pour cela, rendez-vous dans

FLFramework/Config/bootstrap.php

Ici vous pouvez configurer une multitude de chose.

  • cores\Sql:: La connexion à la base de données
  • cores\Router:: Les options par défauts du router
  • cores\Cache:: Les options d'admin du cache
  • cores\Log:: Les options d'admin des logs
  • define('__site_url__', "URL"); L'url spécifique du site si nécessaire
  • App:: Ces fonctions vous permettent de choisir si vous souhaitez utiliser une base de données ou non, le module d'url rewriting ou non...etc.

Modules natifs

Quelques modules sont intégrés nativement dans le framework et permettent certaines choses importantes

FLFramework/Module/DALGenerator

Le DALGenerator va vour permettre de générer vos classes de manipulation de la base de données. Toutes les informations techniques nécessaire sont fournies à cette url :

http://YOUR_URL/Module/DALGenerator/index.html

Cet outil vous permet une totale libertée dans la création de votre base données et dans vos pattern de nommage. En effet, l'application ira chercher toutes vos tables de votre base de données pour en créer des classes miroir implémentant toutes les fonctions de base nécessaire comme :

  • getAll()
  • getById($id)
  • add($object)
  • update($object)
  • delete($id)

De même, ce programme va générer les modèles images de vos tables appelées "Entity"

Il est préférable de stocker ces classes ici :

FLFramework/Data

Puisqu'y résides déjà les trois classes indispensables au routing

 

FLFramework/Module/NativeSQLModule

Ce module est quasi indispensable au bon fonctionnement du framework. Il ajoute à celui-ci les fonctionnalités de réécriture d'url et de multilinguisme. Il vous sera d'ailleurs rappelé (en mode debug) d'importer ces scripts SQL pour votre bien !

Ce module consiste en trois fichiers SQL (un par rôle) à importer dans votre base de données.

Les exceptions natives

Les exceptions natives sont dans le dossiers ci-dessous :

FLFramework/Library/Adapter

Il en existe trois :

  • connexionException
  • controllerException
  • viewException

Elle gère les exceptions basiques du site. (connexion à la BDD impossible, controller introuvable, layout introuvable...etc)

Ces exceptions sont gérées par la classe error.php située dans :

FLFramework/Library/Core

Elles sont désactivables en mettant le mode debug à false ou en réglant les problèmes soulevés.

Les helpers

Les helpers sont des classes encapsulant certaines fonctionnalités utiles à un site web et disponibles dans les controllers ainsi que dans les vues.

Voici les helpers existant

  • FLFramework/Library/Helper/auth.php Ce helper gère la couche authentification de l'utilisateur (connexion/deconnexion/récupération...etc.)
  • FLFramework/Library/Helper/form.php Ce helper gère la couche formulaire (génération automatique, réception d'information...etc.)
  • FLFramework/Library/Helper/mail.php Ce helper gère la couche email (création d'un e-mail, envoie, template...etc.)
  • FLFramework/Library/Helper/rss.php Ce helper permet de générer des flux rss
  • FLFramework/Library/Helper/session.php ce helper gère la couche session

Toutes ces classes sont commentées et très facile d'utilisation.

L'application

Normalement, vous n'aurez à toucher que le dossier :

FLFramework/Website

Ce dossier contient votre application.

  • FLFramework/Website/Content Contient vos assets (javascript, css, media...etc.)
  • FLFramework/Website/MVC Contient votre site web découpé en MVC (Model, Vue, Controleur)
  • FLFramework/Website/Partial Contient les vues partielles
